TARGET_IP = 10.114.152.123

first some basic nmap
nmap -sS 10.114.152.123 -v
nmap -sV -p 80,443 10.114.152.123 -v

we found http on 80 and https on ports 443 and 51337
on the http there was an apache2 default page and the https couldn't be accessed
for some reason I also tried to enum the http (a bit pointless, yes)

gobuster dir -u "http://10.114.152.123/" -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt
as expected, nothing

This looks like a classic problem where:
    we don't have the host in /etc/hosts,
    so we can't access the https site(s)

We will need to try nmap -sC (script scan) for more info (ideally hostname)
nmap -sC -p 80,443 10.114.152.123 -v

we are saved
# ssl-cert: Subject: commonName=grep.thm/organizationName=SearchME
let's add this to /etc/hosts
nano /etc/hosts
add this line
10.114.152.123 grep.thm

now we can access grep.thm, which opens this page:
https://grep.thm/public/html/

we immediatly see a login form (login.php) and a register form (register.php)
we could try OWASP 2025 07: Authentication Failures (eg. register as aDmiN and login as admin)
there is an expired API key so register doesnt work now
