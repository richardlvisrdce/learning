enumerates web directories, DNS subdomains, AWS S3 (Simple Storage Service) buckets, vhosts etc.
by brute force

basic syntax (for directory enumeration):
    gobuster dir -u "URL" -w WORDLIST -t THREADS
    the more THREADS the faster
    don't forget protocol in URL (eg. http://)

you can view mode helps with gobuster MODE --help

subdomain enumeration

what is that?
 - you know website.com
 - you want service.website.com (eg. gemini.google.com)

syntax
 - gobuster dns -d website.com -w WORDLIST

gobuster dns -d offensivetools.thm -w /usr/share/wordlists/SecLists/Discovery/DNS/subdomains-top1million-5000.txt --resolver TARGET_IP
for some reason we are supposed to ignore duplicates with different casing btw

vhost enumeration
syntax
 - gobuster vhost -u "http://website.com" -w WORDLIST

# this command we straight up copied, so I will try to explain
gobuster vhost -u "http://TARGET_IP" --domain offensivetools.thm -w /usr/share/wordlists/SecLists/Discovery/DNS/subdomains-top1million-5000.txt --append-domain --exclude-length 250-320

explanation:
gobuster vhost -u "http://TARGET_IP" # basic syntax
--domain offensivetools.thm # sets top level domain to .thm and second level domain to offensivetools in the Hostname part of our request
-w /usr/share/wordlists/SecLists/Discovery/DNS/subdomains-top1million-5000.txt # a common wordlist for subdomains
--append-domain # this appends offensivetools.thm to each wordlist entry, so that it resolves to a valid hostname
--exclude-length 250-320 # this tries to filter out 404 etc. responses by size-filtering (supposedly doesn't work always but here it does)
